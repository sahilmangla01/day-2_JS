Q.1 What is execution context?

When the JavaScript engine scans a script file, it makes an environment called the Execution Context that handles the entire transformation and execution of the code. During the context runtime, the parser parses the source code and allocates memory for the variables and functions and the scope chain. It is destroyed once the code execution is completed.

Q.2 What is an event loop and call stack?

Event Loop : gives JavaScript its multitasking abilities. This loop constantly check if the call stack is empty or not and if it is then the functions waiting to be executed inside callback queue will be sent over to callstack

Call Stack : used by JavaScript to keep track of the function calls . It is just like the stack data structures and follows the rule of first in last out . We use callstack to memorize which function is running right now.

Q.3 What is creation phase and execution phase?

Creation Phase : involves the creation of global and function execution context, creating the scope chain and allocating the memories to the variables and functions.

Execution Phase : executes the code line by line and stop at the line where we get any error . This includes the ecaluation and the execution of the code.

In summary, the creation phase and execution phase in JavaScript refer to the moment when the variable and function declarations are created and stored in memory, and the moment when the code is actually executed, line by line.

Q.4 What is the spread operator?

Spread Operator : in JavaScript helps to copy the complete or part of an iterable (strings or arrays) as an individual so that we can operate on that data however we want . This operator is denoted by three dots(...) 

 let obj1 = [1 , 2 , 3];
                let obj2 = [3 , 4 , 5];
                let newObj = [...obj1 , ...obj2];
                console.log(newObj)  //Output is [1 , 2 , 3 , 4 , 5 , 6]

Q.7 What is callback hell?

Callback hell is a common issue in JavaScript that arises when dealing with multiple asynchronous operations that depend on each other. It occurs when the code becomes nested within multiple callback functions, making it difficult to read and debug.

function getData(callback){
    console.log("Got the Data ......")
    setTimeout(()=>{
        callback();
    },2000)
}

function processData(callback){
    console.log("Process the Data ......")
    setTimeout(()=>{
        callback();
    },2000)
}

function useData(){
    console.log("use the Data ......")
    
}


getData(()=>{
    processData(()=>{
        useData()
    })
})
